# 魔法链接登录系统 - 完整需求分析文档
---

# 
## 一、项目概述
### 1.1 项目目标
构建一个**极简的邮箱魔法链接登录系统**，支持用户通过邮箱验证码链接登录/注册，支持FREE/PRO身份切换，作为教学演示Demo和后续商业系统的认证模块雏形。

### 1.2 核心原则
+ 代码量最小（总计不超过400行）
+ 零外部依赖（除Redis和SQLite）
+ 单HTML页面完成所有交互
+ 可平滑迁移到生产环境

---

## 二、功能需求
### 2.1 用户登录/注册
| 功能 | 描述 |
| --- | --- |
| 邮箱输入 | 用户输入邮箱地址 |
| 发送魔法链接 | 点击按钮后，系统生成一次性登录链接发送到邮箱 |
| 点击链接登录 | 用户点击邮箱中的链接，自动完成登录/注册 |
| 重新发送 | 支持重新发送魔法链接（生成新token，旧token失效） |


### 2.2 用户状态管理
| 功能 | 描述 |
| --- | --- |
| 登录状态保持 | 用户登录后，刷新页面保持登录状态 |
| 登出 | 点击登出按钮，清除登录状态 |
| 自动过期 | Session 30天无活动自动过期 |


### 2.3 身份系统
| 身份类型 | 标识 | 升级方式 |
| --- | --- | --- |
| FREE | 灰色头像/标识 | 默认身份 |
| PRO | 金色头像/标识 | 通过"模拟支付"手动升级 |


### 2.4 模拟支付（预留接口）
| 功能 | 描述 |
| --- | --- |
| 手动升级 | 登录状态下，输入任意订单号，将当前用户升级为PRO |
| 订单号验证 | 当前版本不做真实验证，仅演示流程 |

## 四、数据存储设计
### 4.1 Redis存储结构
**魔法链接token（15分钟过期）：**

```plain
Key: magic:{32位随机字符串}
Value: 用户邮箱
TTL: 900秒（15分钟）
```

**用户Session（30天过期）：**

```plain
Key: session:{session_id}
Value: user_id (数字)
TTL: 2592000秒（30天）
```

### 4.2 SQLite数据库表结构
**表名：users**

| 字段名 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 用户ID |
| email | TEXT | UNIQUE NOT NULL | 用户邮箱 |
| is_pro | INTEGER | DEFAULT 0 | 是否PRO（0=FREE,1=PRO） |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 注册时间 |
| last_login_at | TIMESTAMP |  | 最后登录时间 |

## 五、API接口规范
### 5.1 发送魔法链接
**请求：**

```plain
POST /api/auth/send-magic-link
Content-Type: application/json

{
    "email": "user@example.com"
}
```

**响应（成功）：**

```json
{
    "success": true,
    "message": "魔法链接已发送，请查收邮件"
}
```

**响应（失败）：**

```json
{
    "success": false,
    "message": "邮箱格式错误"
}
```

**错误码：**

+ 400: 邮箱格式错误
+ 429: 发送过于频繁（可选，MVP可暂不实现）

**后端逻辑：**

1. 验证邮箱格式
2. 生成32位随机字符串作为token
3. 存入Redis：`magic:{token}` = email，设置15分钟过期
4. 调用Resend API发送邮件
5. 邮件内容：包含链接 `http://localhost:8000/api/auth/verify?token={token}`

---

### 5.2 验证魔法链接（用户点击链接后）
**请求：**

```plain
GET /api/auth/verify?token={token}
```

**响应（成功）：**

+ 状态码：302 重定向到首页 `/`
+ 设置Cookie：`session_id={session_id}`

**响应（失败）：**

+ 返回HTML错误页面（或JSON错误，前端自行跳转）

```json
{
    "success": false,
    "message": "链接无效或已过期"
}
```

**后端逻辑：**

1. 从参数获取token
2. 从Redis获取 `magic:{token}` 对应的email
3. 如果不存在或已过期 → 返回错误
4. 从SQLite查找或创建用户
5. 生成唯一session_id（可使用uuid）
6. 存入Redis：`session:{session_id}` = user_id，设置30天过期
7. 设置Cookie：`session_id={session_id}`，HttpOnly=True
8. 删除Redis中的魔法链接token（`magic:{token}`）
9. 重定向到首页 `/`

---

### 5.3 获取当前用户信息
**请求：**

```plain
GET /api/user/me
Cookie: session_id={session_id}
```

**响应（已登录）：**

```json
{
    "success": true,
    "user": {
        "id": 1,
        "email": "user@example.com",
        "is_pro": true,
        "created_at": "2024-01-01T00:00:00"
    }
}
```

**响应（未登录）：**

```json
{
    "success": false,
    "message": "未登录"
}
```

**后端逻辑：**

1. 从Cookie获取session_id
2. 从Redis获取 `session:{session_id}` 对应的user_id
3. 如果不存在 → 返回未登录
4. 从SQLite查询用户信息
5. 刷新Redis中session的过期时间（重置为30天）
6. 返回用户信息（不含密码等敏感字段）

---

### 5.4 升级为PRO（模拟支付）
**请求：**

```plain
POST /api/user/upgrade-to-pro
Cookie: session_id={session_id}
Content-Type: application/json

{
    "order_id": "任意字符串（仅用于演示）"
}
```

**响应（成功）：**

```json
{
    "success": true,
    "is_pro": true,
    "message": "升级成功，您现在是PRO用户了"
}
```

**响应（失败）：**

```json
{
    "success": false,
    "message": "请先登录"
}
```

**后端逻辑：**

1. 从Cookie获取session_id
2. 从Redis获取user_id
3. 验证用户已登录
4. 更新SQLite：`UPDATE users SET is_pro = 1 WHERE id = {user_id}`
5. 返回成功响应（订单号当前版本仅记录日志，不做验证）

---

### 5.5 登出
**请求：**

```plain
POST /api/auth/logout
Cookie: session_id={session_id}
```

**响应（成功）：**

```json
{
    "success": true,
    "message": "已登出"
}
```

**后端逻辑：**

1. 从Cookie获取session_id
2. 从Redis删除 `session:{session_id}`
3. 返回成功响应

---

### 5.6 静态页面
**请求：**

```plain
GET /
```

**响应：**

+ 返回 `static/index.html` 文件（HTML内容）

## 六、前端界面设计要求
### 6.1 技术约束
+ 单HTML文件，内联CSS和JavaScript
+ 不使用任何前端框架（Vue/React等）
+ 使用Fetch API调用后端接口
+ 页面根据登录状态动态切换视图（无需刷新）

### 6.2 视图一：未登录状态
**UI元素：**

+ 标题："魔法链接登录"
+ 邮箱输入框（type=email，必填）
+ 发送按钮："发送魔法链接"
+ 提示信息区域

**交互流程：**

1. 用户输入邮箱，点击发送按钮
2. 前端调用 POST `/api/auth/send-magic-link`
3. 成功后显示："验证链接已发送到您的邮箱，请查收"
4. 失败后显示错误信息

### 6.3 视图二：已登录状态（FREE用户）
**UI元素：**

+ 欢迎信息："欢迎，{email}"
+ 身份标识：灰色圆形或灰色文字"FREE用户"
+ 模拟支付区域：
    - 订单号输入框（placeholder="输入任意订单号"）
    - 升级按钮："升级为PRO"
+ 登出按钮

**交互流程：**

1. 页面加载时调用 GET `/api/user/me` 获取用户信息
2. 显示用户邮箱和身份状态
3. 点击升级按钮 → 调用 POST `/api/user/upgrade-to-pro`
4. 成功后刷新用户信息（重新调用 `/api/user/me`），显示金色身份
5. 点击登出 → 调用 POST `/api/auth/logout`，切换回未登录视图

### 6.4 视图三：已登录状态（PRO用户）
**UI元素：**

+ 欢迎信息："欢迎，{email}"
+ 身份标识：金色圆形或金色文字"✨ PRO用户 ✨"
+ 额外显示："🎉 感谢支持！已永久激活PRO"
+ 登出按钮

**交互流程：**

+ 同FREE用户，但无模拟支付区域

### 6.5 页面打开时的初始化逻辑
1. 调用 GET `/api/user/me`
2. 如果返回用户信息 → 显示已登录视图
3. 如果返回未登录 → 显示未登录视图

## 十、代码约束
### 10.1 代码行数要求
+ 每个文件不超过150行
+ 总代码量控制在400行以内

### 10.2 函数划分建议
**main.py 包含：**

+ FastAPI应用初始化
+ 路由端点（6个API + 1个静态页面路由）
+ Session验证的依赖注入函数 `get_current_user()`

**database.py 包含：**

+ 数据库初始化函数 `init_db()`
+ 查找或创建用户 `get_or_create_user(email)`
+ 根据ID获取用户 `get_user_by_id(user_id)`
+ 升级用户为PRO `upgrade_to_pro(user_id)`
+ 更新最后登录时间 `update_last_login(user_id)`

**redis_client.py 包含：**

+ Redis连接获取 `get_redis()`
+ 创建魔法链接token `create_magic_token(email)`
+ 验证魔法链接token `verify_magic_token(token)`
+ 创建Session `create_session(user_id)`
+ 获取Session `get_session(session_id)`
+ 删除Session `delete_session(session_id)`
+ 刷新Session过期时间 `refresh_session(session_id)`

### 10.3 错误处理
+ 所有API返回统一的JSON格式（含success字段）
+ HTTP状态码使用标准语义（200成功，400客户端错误，401未授权）
+ 数据库/Redis操作异常需捕获并返回友好错误信息
