

## 一、系统基础配置

```bash
# 查看网络状态
ip addr

# 查看 Python 版本
python3 -V

# 安装 git
yum install -y git

# 关闭防火墙（解决 Windows 访问问题）
systemctl stop firewalld

# 开放端口（备用方案）
firewall-cmd --add-port=8000/tcp --permanent
firewall-cmd --reload
```

## 二、项目部署

```bash
# 创建目录
mkdir student

# 进入目录
cd student

# 查看项目文件,.env是隐藏，一定要查看全部
ls -a
ll
#windows传输项目文件到Linux，使用Linux的IP和用户名。在windows的终端进行操作。
scp -r "D:\deepseek-api-oula" root@192.168.152.133:/root/student/

# 回Linux查看是否完成传输，模板目录是否完整
ls -a
cd

# 安装 Python 依赖（全部成功）
pip3 install -r requirements-demo.txt

# 验证安装
pip3 list | grep -E "fastapi|uvicorn|jinja2|python-multipart|openai|python-dotenv|httpx"

# 编辑环境变量文件
vim .env

# 启动服务（成功）
uvicorn app:app --host 0.0.0.0 --port 8000 --reload
```

## 五、Windows 访问验证步骤

1. **查看虚拟机实际 IP**：
```bash
ip addr show ens33 | grep inet
```

2. **Windows 浏览器访问**：
```
http://<虚拟机IP>:8000
```

3. **如果还是无法访问**，在虚拟机内测试：
```bash
curl http://127.0.0.1:8000
```
能通说明服务正常，问题在防火墙或网络配置。

**sh 脚本不可用的原因**：可能是脚本格式问题（Windows 换行符 `\r\n` vs Linux `\n`），或缺少执行依赖。但手动执行 `uvicorn` 命令已成功启动，不影响使用。
