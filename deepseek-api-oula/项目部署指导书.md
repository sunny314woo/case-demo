# DeepSeek FastAPI 教学 Demo 项目部署指导书

## 1. 文档用途

本文档用于指导教师或学生在 openEuler 22.03 命令行环境中，从零开始完成本项目的部署、启动与访问。

本文档适用于以下场景：

1. 教师课前搭建演示环境
2. 学生机房实验部署
3. 教学展示前的环境检查

## 2. 项目简介

本项目是一个基于 FastAPI 的教学 Demo，用于演示如何调用 DeepSeek API，实现：

1. 文本改写
2. 风格生成
3. Prompt 展示与编辑

项目特点：

1. 不使用数据库
2. 不使用 nginx
3. 不使用前端框架
4. 代码结构简单，适合课堂讲解

## 3. 部署目标

完成部署后，应达到以下效果：

1. openEuler 虚拟机中能够正常启动 FastAPI 服务
2. Windows 浏览器能够访问项目页面
3. 页面能够调用 DeepSeek API 返回结果

## 4. 环境要求

### 4.1 操作系统

1. openEuler 22.03

### 4.2 Python 环境

1. Python 3.9 或可用的 Python 3
2. windows安装依赖，运行网页服务程序。
   1. 如window，pycharm可以采用：pip install -r requirements-demo.txt
   2. window 安装完成程序后调试：D:\...\deepseek-api-oula> uvicorn app:app --reload
   3. 取消程序运行Ctrl+C
   
   




### 4.3 网络环境

1. VMware 虚拟机网络建议使用桥接模式
2. 虚拟机需能访问互联网
3. Windows 主机与虚拟机需处于可互通网络环境

### 4.4 API 条件

1. 需要准备可用的 DeepSeek API Key

## 6. 部署前准备

在开始部署前，请确认以下事项：

1. 虚拟机已经启动
2. 能正常登录 openEuler 命令行
3. 已知虚拟机用户名和 IP 地址
4. Windows 主机可以使用 `scp` 传文件到虚拟机

## 7. 第一步：获取项目文件

### 7.1 在本机查看项目目录

假设你当前项目目录为：

```text
/Users/wufengyu/Projects/deepseek-api-oula
```

### 7.2 使用 scp 传到 openEuler

在本机终端执行：

```bash
scp -r /Users/wufengyu/Projects/deepseek-api-oula 用户名@虚拟机IP:/home/用户名/
```

示例：

```bash
scp -r /Users/wufengyu/Projects/deepseek-api-oula student@192.168.1.20:/home/student/
```

传输完成后，项目目录会出现在 openEuler 的用户主目录中。

## 8. 第二步：登录 openEuler

通过终端登录虚拟机：

```bash
ssh 用户名@虚拟机IP
```

示例：

```bash
ssh student@192.168.1.20
```

登录后进入项目目录：

```bash
cd /home/用户名/deepseek-api-oula
```

示例：

```bash
cd /home/student/deepseek-api-oula
```

## 9. 第三步：查看项目文件是否完整

执行：

```bash
ls
```

应至少看到以下文件：

```text
app.py
install_demo_prereqs.sh
start_demo.sh
requirements-demo.txt
templates
```

如果 `templates` 目录缺失，页面将无法正常渲染。

## 10. 第四步：安装前置依赖

### 10.1 给脚本执行权限

执行：

```bash
chmod +x install_demo_prereqs.sh
```

### 10.2 执行安装脚本

如果希望把虚拟环境安装到当前项目目录，执行：

```bash
./install_demo_prereqs.sh .
```

这里的 `.` 表示“当前目录”。

脚本会自动完成以下工作：

1. 检查 `dnf` 或 `yum`
2. 安装 Python、pip、venv 等基础依赖
3. 创建 `.venv` 虚拟环境
4. 使用国内 pip 镜像安装 Python 包
5. 显式使用虚拟环境中的 Python 安装依赖，避免系统 `python` 命令差异导致失败

### 10.3 安装成功后的检查

执行：

```bash
ls -a
```

如果看到 `.venv`，说明虚拟环境已经创建成功。

## 11. 第五步：配置 DeepSeek API Key

### 11.1 复制配置模板

执行：

```bash
cp .env .env
```

### 11.2 编辑配置文件

如果系统中有 `vi`，执行：

```bash
vi .env
```

将下面这一行：

```text
DEEPSEEK_API_KEY=sk-请替换成你的Key
```

改成你自己的真实 Key，例如：

```text
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxx
```

同时保留下面两项默认配置：

```text
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-chat
```

### 11.3 配置说明

各项配置含义如下：

1. `DEEPSEEK_API_KEY`：调用 DeepSeek 接口的密钥
2. `DEEPSEEK_BASE_URL`：DeepSeek 接口地址
3. `DEEPSEEK_MODEL`：默认使用的模型名称

## 12. 第六步：启动项目

### 12.1 给启动脚本执行权限

执行：

```bash
chmod +x start_demo.sh
```

### 12.2 启动项目

执行：

```bash
./start_demo.sh
```

正常情况下，你会看到类似输出：

```text
正在启动 FastAPI Demo ...
浏览器访问地址： http://服务器IP:8000
INFO:     Uvicorn running on http://0.0.0.0:8000
```

这表示服务已经启动成功。

说明：

1. 默认采用稳定演示模式启动，不启用 `--reload`
2. 这样更适合课堂环境，避免因文件变化导致服务重启
3. 如果只是在教师机器上本地调试，可手动执行带 `--reload` 的命令

## 13. 第七步：查看虚拟机 IP

如果你不确定虚拟机 IP，可以在 openEuler 中执行：

```bash
ip addr
```

或执行：

```bash
hostname -I
```

记下网卡对应的 IPv4 地址，例如：

```text
192.168.1.20
```

## 14. 第八步：在 Windows 浏览器访问项目

在 Windows 主机浏览器地址栏中输入：

```text
http://虚拟机IP:8000
```

示例：

```text
http://192.168.1.20:8000
```

如果页面成功打开，说明部署完成。

## 15. 页面操作说明

项目页面主要包含以下区域：

1. 功能模式选择
2. 风格选择
3. 原始文本输入框
4. Prompt 展示与编辑框
5. 调用 AI 生成按钮
6. 输出结果显示框

建议课堂演示流程：

1. 保持默认 Prompt 先生成一次
2. 改变风格后再生成一次
3. 手动修改 Prompt 中一条要求后再生成一次
4. 对比结果变化

## 16. 停止项目

如果当前项目正在前台运行，可按：

```bash
Ctrl + C
```

停止服务。

## 17. 重新启动项目

下次进入项目目录后，只需要执行：

```bash
./start_demo.sh
```

一般不需要重复安装依赖，除非你删除了 `.venv`。

## 18. 常见问题与参考解决办法

### 问题 1：执行安装脚本时报 “Permission denied”

可能原因：

1. 脚本没有执行权限

解决方法：

```bash
chmod +x install_demo_prereqs.sh
./install_demo_prereqs.sh .
```

### 问题 2：提示找不到 `dnf` 或 `yum`

可能原因：

1. 当前系统不是标准 openEuler 环境
2. 软件包管理工具未正确安装

解决方法：

1. 先确认系统版本
2. 检查是否为 openEuler 22.03
3. 请管理员修复系统包管理工具后再继续部署

### 问题 3：安装 Python 依赖时很慢或失败

可能原因：

1. 网络不稳定
2. 默认镜像访问慢

解决方法：

可在执行安装脚本时指定镜像源，例如：

```bash
PIP_INDEX_URL=https://mirrors.aliyun.com/pypi/simple ./install_demo_prereqs.sh .
```

### 问题 4：启动时提示未找到 `.venv`

可能原因：

1. 还没有执行安装脚本
2. 安装过程失败
3. 当前不在项目目录中

解决方法：

1. 先执行 `pwd` 确认当前目录
2. 确认目录中存在 `install_demo_prereqs.sh`
3. 重新执行：

```bash
./install_demo_prereqs.sh .
```

### 问题 5：启动后浏览器打不开页面

可能原因：

1. 服务没有真正启动成功
2. IP 地址写错
3. 虚拟机网络未配置好
4. 防火墙拦截了 8000 端口

解决方法：

1. 在 openEuler 中查看终端是否出现 `Uvicorn running on http://0.0.0.0:8000`
2. 用 `hostname -I` 查看虚拟机真实 IP
3. 确认 VMware 使用桥接模式
4. 在虚拟机本机先测试：

```bash
curl http://127.0.0.1:8000
```

如果本机可访问、Windows 不可访问，再重点检查网络和防火墙。

### 问题 6：页面打开了，但点击生成后提示未配置 API Key

可能原因：

1. `.env` 文件未创建
2. `.env` 文件中没有填写真实 Key
3. Key 填写格式错误

解决方法：

1. 确认当前目录中存在 `.env`
2. 执行：

```bash
cat .env
```

3. 检查 `DEEPSEEK_API_KEY` 是否已经替换成真实密钥

补充说明：

1. 当前启动脚本会在启动时主动提示 API Key 是否为空
2. 即使 Key 未配置，页面也能打开，但 AI 生成不会成功

### 问题 7：页面打开了，但调用失败

可能原因：

1. Key 无效
2. 网络无法访问 DeepSeek 接口
3. 模型名配置错误
4. 当前网络环境限制外部 API 访问

解决方法：

1. 检查 `.env` 中的 Key 是否正确
2. 保持默认：

```text
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-chat
```

3. 检查虚拟机是否能联网
4. 必要时在虚拟机中测试网络连通性：

```bash
curl https://api.deepseek.com
```

### 问题 8：修改了 `.env`，但程序好像没有生效

可能原因：

1. 程序启动后才修改 `.env`
2. 服务未重启

解决方法：

1. 先按 `Ctrl + C` 停止项目
2. 重新执行：

```bash
./start_demo.sh
```

如果你使用的是手动调试命令，也同样需要重启 `uvicorn` 进程。

### 问题 9：页面样式正常，但没有生成结果

可能原因：

1. Prompt 为空
2. 接口调用异常
3. 模型返回内容为空

解决方法：

1. 先点击“重置为默认 Prompt”
2. 再重新生成
3. 观察页面错误提示内容

### 问题 10：学生误改 Prompt 后结果很奇怪

可能原因：

1. Prompt 被改坏了
2. 要求不清晰或互相冲突

解决方法：

1. 点击“重置为默认 Prompt”
2. 引导学生一次只改一条要求
3. 对比修改前后的输出差异，作为课堂讨论内容

## 19. 部署检查清单

课前建议按下面清单快速检查一次：

1. 项目目录是否完整
2. `.venv` 是否存在
3. `.env` 是否已配置真实 API Key

5. `hostname -I` 是否能看到可访问 IP
6. Windows 浏览器是否能打开页面
7. 点击生成是否能返回结果
